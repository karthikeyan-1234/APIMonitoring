1. Overview & Flow

This guide implements a modern OTLP Push → Collector → Prometheus Scrape observability pipeline.

Clear Breakdown (Push vs Pull)

StepComponentActionDirectionProtocol / PortWhat Happens
1.NET 9 Web APIPushAPI → CollectorOTLP gRPC 4317Auto-instrumented metrics + resource attributes
2OpenTelemetry CollectorReceive + Process——batch + k8sattributes + label conversion
3OpenTelemetry CollectorExpose—HTTP 8889Prometheus-compatible metrics endpoint
4PrometheusScrape (Pull)Collector → PrometheusHTTP every 5sStores enriched metrics
5GrafanaQuery & VisualizeGrafana → Prometheus—Dashboards with version labels

Why This Hybrid Model?

Push from the app is lightweight and reliable for .NET. Pull by Prometheus is its native strength. The Collector provides central enrichment and automatic conversion of OTEL resource attributes into Prometheus labels.

  1. Save all YAML files below into individual files
  2. Apply in this order: OTEL Collector files → Prometheus files → Grafana files → API Deployments
  3. Wait for all pods to reach Ready state
  4. Access Prometheus via NodePort 30090 and Grafana via your ingress

5. .NET 9 Web API Instrumentation

Click to expand: Full Program.cs (generalized for any .NET 9 Web API) 
using OpenTelemetry.Logs;
using OpenTelemetry.Metrics;
using OpenTelemetry.Resources;
using OpenTelemetry.Trace;

var builder = WebApplication.CreateBuilder(args);

var serviceName = "YOUR_SERVICE_NAME"; // Change to your service name
var serviceVersion = Environment.GetEnvironmentVariable("APP_VERSION") ?? "1.0.0";

Action<ResourceBuilder> configureResource = r => r
    .AddService(serviceName: serviceName, serviceVersion: serviceVersion)
    .AddAttributes(new Dictionary<string, object> { ["service.version"] = serviceVersion });

builder.Logging.ClearProviders();
builder.Logging.AddOpenTelemetry(options => {
    options.SetResourceBuilder(ResourceBuilder.CreateDefault().AddService(serviceName, serviceVersion));
    options.AddOtlpExporter(o => o.Endpoint = new Uri("http://otel-collector:4317"));
});

builder.Services.AddOpenTelemetry()
    .ConfigureResource(configureResource)
    .WithTracing(tracing => tracing.AddAspNetCoreInstrumentation().AddHttpClientInstrumentation().AddOtlpExporter(o => o.Endpoint = new Uri("http://otel-collector:4317")))
    .WithMetrics(metrics => metrics.AddAspNetCoreInstrumentation().AddHttpClientInstrumentation().AddRuntimeInstrumentation().AddOtlpExporter(o => o.Endpoint = new Uri("http://otel-collector:4317")));

var app = builder.Build();
app.MapGet("/test", () => Results.Ok(new { Message = "Telemetry working!" })); // Used by liveness/readiness probes
app.Run();

9. Troubleshooting & Edge Cases

  • No metrics in Grafana: Check otel-collector logs and Prometheus targets page.
  • Missing service.version label: Verify APP_VERSION environment variable and resource_to_telemetry_conversion in otel-collector-config.yaml.
  • Collector unreachable: Ensure pods are in the same namespace and DNS resolution works.
  • High cardinality: k8sattributes adds many labels — consider Prometheus relabeling if needed.
