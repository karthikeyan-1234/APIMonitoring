Header-Based Traffic Routing (Industry Standard)
The safest and most widely used production routing pattern across modern API gateways.
Core Idea: Add a higher-priority route with a header condition. Do NOT modify existing routes.

The Core Principle

/test (priority 0) → stable
+
/test + X-Canary=true (priority 10) → canary
Kong evaluates routes by regex_priority. Higher priority wins first.

Why This is Production Safe

No route changes
No service changes
Zero downtime
Instant rollback

Implementation — Canary

# Create canary service
Invoke-RestMethod -Uri "http://kong-admin.local/services" `
-Method POST `
-Body (@{
  name="api1-canary-direct"
  host="api1-canary-service.default.svc.cluster.local"
  port=80
} | ConvertTo-Json)
# Create header route
Invoke-RestMethod -Uri "http://kong-admin.local/services/api1-canary-direct/routes" `
-Method POST `
-Body (@{
  name="api1-canary-header-route"
  paths=@("/test")
  headers=@{ "X-Canary"=@("true") }
  regex_priority=10
} | ConvertTo-Json)

Verification

# Stable
Invoke-RestMethod http://kong-proxy.local/test

# Canary
Invoke-RestMethod http://kong-proxy.local/test `
-Headers @{ "X-Canary"="true" }

Blue/Green (Header Based)

# Green service
Invoke-RestMethod -Uri "http://kong-admin.local/services" `
-Method POST `
-Body (@{
  name="api1-green-direct"
  host="api1-green-service.default.svc.cluster.local"
  port=80
} | ConvertTo-Json)
# Route
Invoke-RestMethod -Uri "http://kong-admin.local/services/api1-green-direct/routes" `
-Method POST `
-Body (@{
  paths=@("/bgtest")
  headers=@{ "X-Slot"=@("green") }
  regex_priority=10
} | ConvertTo-Json)

Header Conventions

HeaderUse
X-CanaryCanary routing
X-VersionVersion routing
X-User-GroupInternal QA
X-Feature-FlagFeature toggle
X-SlotBlue/Green

Rollback

Invoke-RestMethod `
-Uri "http://kong-admin.local/routes/api1-canary-header-route" `
-Method DELETE
Delete the route → instantly back to stable. No downtime.
